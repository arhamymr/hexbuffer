## Overview

A **Man-in-the-Middle (MITM) proxy** sits transparently between a client and a server. It:

- **Terminates** the client's TLS connection using a dynamically generated fake certificate
- **Decrypts**, reads, and optionally **modifies** the plaintext HTTP traffic
- **Re-encrypts** and forwards the traffic to the real server over a new TLS connection

Rust is particularly well-suited for this because of:

- Zero-cost async I/O via `tokio`
- Pure-Rust TLS via `rustls` (no OpenSSL dependency, memory-safe)
- Ergonomic X.509 certificate generation via `rcgen`
- High-performance HTTP parsing via `httparse` or `hyper`

---

## How MITM Works

```
Client ──[TLS with fake cert]──► MITM Proxy ──[TLS with real cert]──► Real Server
                                     │
                              Decrypted plaintext
                              (can read/modify)
```

The client is deceived because the MITM proxy presents a **fake certificate** signed by a **root CA that the client trusts**. This root CA is generated by you and must be installed in the client's trust store ahead of time.

---

## Attack Flow Diagram

```
1. Client sends HTTP CONNECT to proxy
   CONNECT example.com:443 HTTP/1.1

2. Proxy replies HTTP 200
   HTTP/1.1 200 Connection Established

3. Proxy generates fake cert for "example.com" on-the-fly
   (signed by attacker's root CA)

4. Proxy performs TLS handshake WITH CLIENT using fake cert
   Client accepts it (trusts attacker's CA)

5. Proxy connects to real example.com:443
   Normal TLS handshake — proxy trusts Mozilla CA bundle

6. Proxy now sits in the middle:
   Client <──[decrypted]──> Proxy <──[decrypted]──> Server
```

## Layer 1 — Async Runtime (tokio)

`tokio` provides the async executor, TCP primitives, and task spawning. Each incoming connection gets its own `tokio::spawn` task — lightweight green threads, not OS threads.

### Basic TCP accept loop

```rust
use tokio::net::TcpListener;
use anyhow::Result;

#[tokio::main]
async fn main() -> Result<()> {
    tracing_subscriber::fmt::init();

    let listener = TcpListener::bind("127.0.0.1:8080").await?;
    tracing::info!("Proxy listening on 127.0.0.1:8080");

    loop {
        let (stream, peer_addr) = listener.accept().await?;
        tracing::debug!("New connection from {peer_addr}");

        tokio::spawn(async move {
            if let Err(e) = handle_connection(stream).await {
                tracing::error!("Connection error: {e}");
            }
        });
    }
}
```

### Key tokio types used

| Type | Purpose |
|---|---|
| `TcpListener` | Accept incoming TCP connections |
| `TcpStream` | Bidirectional byte stream |
| `tokio::spawn` | Spawn a non-blocking concurrent task |
| `tokio::io::copy` | Async byte-copy between streams |
| `tokio::io::split` | Split a stream into read/write halves |

## Layer 2 — TLS Termination (rustls + tokio-rustls)

`rustls` is a memory-safe TLS 1.2/1.3 implementation in pure Rust. `tokio-rustls` wraps it with async I/O via `AsyncRead`/`AsyncWrite`.

In a MITM proxy you create **two** TLS configurations per connection:

1. **`ServerConfig`** — used when accepting the client's TLS handshake (you act as a "server")
2. **`ClientConfig`** — used when connecting upstream to the real server (you act as a "client")

### Building a TLS ServerConfig from a generated certificate

```rust
use rustls::{ServerConfig, pki_types::{CertificateDer, PrivateKeyDer}};
use tokio_rustls::TlsAcceptor;
use std::sync::Arc;

fn build_server_config(
    cert_der: Vec<u8>,
    key_der:  Vec<u8>,
) -> Result<ServerConfig> {
    let cert = CertificateDer::from(cert_der);
    let key  = PrivateKeyDer::try_from(key_der)
        .map_err(|e| anyhow::anyhow!("Invalid private key: {e}"))?;

    let config = ServerConfig::builder()
        .with_no_client_auth()
        .with_single_cert(vec![cert], key)?;

    Ok(config)
}
```

### Accepting a TLS connection from a client

```rust
use tokio_rustls::TlsAcceptor;
use tokio::net::TcpStream;

async fn accept_tls(
    stream:        TcpStream,
    server_config: Arc<ServerConfig>,
) -> Result<tokio_rustls::server::TlsStream<TcpStream>> {
    let acceptor = TlsAcceptor::from(server_config);
    let tls_stream = acceptor.accept(stream).await?;
    Ok(tls_stream)
}
```

### Building a TLS ClientConfig (for upstream)

```rust
use rustls::{ClientConfig, RootCertStore};
use tokio_rustls::TlsConnector;
use rustls::pki_types::ServerName;
use webpki_roots::TLS_SERVER_ROOTS;

fn build_client_config() -> Arc<ClientConfig> {
    let mut roots = RootCertStore::empty();
    roots.extend(TLS_SERVER_ROOTS.iter().cloned());

    let config = ClientConfig::builder()
        .with_root_certificates(roots)
        .with_no_client_auth();

    Arc::new(config)
}

async fn connect_upstream_tls(
    stream:        tokio::net::TcpStream,
    hostname:      &str,
    client_config: Arc<ClientConfig>,
) -> Result<tokio_rustls::client::TlsStream<tokio::net::TcpStream>> {
    let connector   = TlsConnector::from(client_config);
    let server_name = ServerName::try_from(hostname.to_string())?;
    let tls_stream  = connector.connect(server_name, stream).await?;
    Ok(tls_stream)
}
```

## Layer 3 — Dynamic Certificate Generation (rcgen)

This is the **core deception mechanism**. You generate a root CA once, install it in the client's trust store, then sign a unique leaf certificate for every hostname on demand.

### Step 1 — Generate the root CA (do this once at startup)

```rust
use rcgen::{Certificate, CertificateParams, DistinguishedName, IsCa, BasicConstraints, KeyPair};
use rcgen::KeyUsagePurpose;

pub struct CertificateAuthority {
    pub cert: Certificate,
    pub key:  KeyPair,
}

impl CertificateAuthority {
    pub fn new() -> Result<Self> {
        let mut params = CertificateParams::default();

        params.is_ca = IsCa::Ca(BasicConstraints::Unconstrained);
        params.key_usages = vec![
            KeyUsagePurpose::KeyCertSign,
            KeyUsagePurpose::CrlSign,
        ];

        let mut dn = DistinguishedName::new();
        dn.push(rcgen::DnType::CommonName, "MITM Root CA");
        dn.push(rcgen::DnType::OrganizationName, "Security Research");
        params.distinguished_name = dn;

        let key  = KeyPair::generate()?;
        let cert = params.self_signed(&key)?;

        Ok(Self { cert, key })
    }

    /// Export root CA as PEM — install this in the client's trust store
    pub fn ca_cert_pem(&self) -> String {
        self.cert.pem()
    }
}
```

### Step 2 — Generate a leaf certificate per hostname

```rust
use rcgen::{CertificateParams, DistinguishedName, KeyPair, SanType};
use std::time::Duration;
use std::time::SystemTime;

pub fn generate_leaf_cert(
    hostname: &str,
    ca: &CertificateAuthority,
) -> Result<(Vec<u8>, Vec<u8>)> {
    // Build SANs: support both exact hostname and wildcard
    let mut params = CertificateParams::new(vec![
        hostname.to_string(),
        format!("*.{}", hostname),
    ])?;

    let mut dn = DistinguishedName::new();
    dn.push(rcgen::DnType::CommonName, hostname);
    params.distinguished_name = dn;

    // Valid for 1 year from now
    params.not_before = rcgen::date_time_ymd(2024, 1, 1);
    params.not_after  = rcgen::date_time_ymd(2025, 1, 1);

    let leaf_key  = KeyPair::generate()?;
    let leaf_cert = params.signed_by(&leaf_key, &ca.cert, &ca.key)?;

    // Return DER-encoded cert and key bytes
    let cert_der = leaf_cert.der().to_vec();
    let key_der  = leaf_key.serialize_der();

    Ok((cert_der, key_der))
}
```

### Step 3 — Cache generated certs (avoid re-generating for the same host)

```rust
use std::collections::HashMap;
use tokio::sync::Mutex;
use std::sync::Arc;

pub struct CertCache {
    inner: Mutex<HashMap<String, (Vec<u8>, Vec<u8>)>>,
}

impl CertCache {
    pub fn new() -> Arc<Self> {
        Arc::new(Self { inner: Mutex::new(HashMap::new()) })
    }

    pub async fn get_or_generate(
        &self,
        hostname: &str,
        ca: &CertificateAuthority,
    ) -> Result<(Vec<u8>, Vec<u8>)> {
        let mut map = self.inner.lock().await;
        if let Some(cached) = map.get(hostname) {
            return Ok(cached.clone());
        }
        let pair = generate_leaf_cert(hostname, ca)?;
        map.insert(hostname.to_string(), pair.clone());
        Ok(pair)
    }
}
```

## Layer 4 — HTTP Parsing (httparse / hyper)

### Option A — `httparse` (zero-copy, low-level)

`httparse` is a streaming, allocation-free parser. It's ideal when you want maximum control and minimum overhead.

```rust
use httparse::{Request, Response, EMPTY_HEADER};

/// Parse an HTTP/1.1 request from a raw buffer
fn parse_request(buf: &[u8]) -> Result<()> {
    let mut headers = [EMPTY_HEADER; 64];
    let mut req     = Request::new(&mut headers);

    match req.parse(buf)? {
        httparse::Status::Complete(n) => {
            println!("Parsed {} bytes", n);
            println!("Method:  {:?}", req.method.unwrap_or(""));
            println!("Path:    {:?}", req.path.unwrap_or(""));
            println!("Version: HTTP/1.{}", req.version.unwrap_or(0));

            for h in req.headers.iter() {
                println!("Header: {}: {}", h.name,
                    std::str::from_utf8(h.value).unwrap_or("<binary>"));
            }
        }
        httparse::Status::Partial => {
            println!("Incomplete — need more data");
        }
    }
    Ok(())
}

/// Parse an HTTP/1.1 response
fn parse_response(buf: &[u8]) -> Result<()> {
    let mut headers = [EMPTY_HEADER; 64];
    let mut res     = Response::new(&mut headers);

    if let httparse::Status::Complete(_) = res.parse(buf)? {
        println!("Status: {}", res.code.unwrap_or(0));
        println!("Reason: {}", res.reason.unwrap_or(""));
    }
    Ok(())
}
```

### Option B — `hyper` (high-level, structured)

`hyper` gives you structured `Request<Body>` / `Response<Body>` types and handles connection keep-alive, chunked encoding, and HTTP/2 automatically.

```rust
use hyper::{Request, Response, body::Incoming};
use hyper::service::service_fn;
use hyper_util::rt::TokioIo;
use http_body_util::{BodyExt, Full};
use bytes::Bytes;

/// A hyper service that intercepts every request
async fn intercept(req: Request<Incoming>) -> Result<Response<Full<Bytes>>, hyper::Error> {
    // Log request details
    tracing::info!(
        method  = %req.method(),
        uri     = %req.uri(),
        version = ?req.version(),
        "Intercepted request"
    );

    // Print all headers
    for (name, value) in req.headers() {
        tracing::debug!("{}: {:?}", name, value);
    }

    // Collect the full body
    let (parts, body)  = req.into_parts();
    let body_bytes     = body.collect().await?.to_bytes();
    tracing::debug!("Body ({} bytes): {:?}", body_bytes.len(), &body_bytes[..body_bytes.len().min(256)]);

    // Reconstruct the request and forward it upstream
    let modified_req = Request::from_parts(parts, body_bytes.clone());
    // ... forward modified_req to upstream ...

    // Return a synthetic response for demonstration
    Ok(Response::new(Full::new(Bytes::from("Intercepted!"))))
}

/// Serve hyper on an already-established TLS stream
async fn serve_http<S>(tls_stream: S) -> Result<()>
where
    S: tokio::io::AsyncRead + tokio::io::AsyncWrite + Unpin + Send + 'static,
{
    let io = TokioIo::new(tls_stream);
    hyper::server::conn::http1::Builder::new()
        .serve_connection(io, service_fn(intercept))
        .await?;
    Ok(())
}
```

## Layer 5 — CONNECT Tunnel Handling

HTTPS proxies use the HTTP `CONNECT` method. The client sends a plaintext `CONNECT` request first, and the proxy responds with `200 Connection Established` before any TLS occurs.

```rust
use tokio::net::TcpStream;
use tokio::io::{AsyncReadExt, AsyncWriteExt};

/// Read the initial CONNECT line from a raw TCP stream
async fn read_connect_target(stream: &mut TcpStream) -> Result<String> {
    let mut buf = vec![0u8; 4096];
    let n = stream.read(&mut buf).await?;
    let raw = std::str::from_utf8(&buf[..n])?;

    // Parse: "CONNECT example.com:443 HTTP/1.1\r\n..."
    let first_line = raw.lines().next().unwrap_or("");
    let parts: Vec<&str> = first_line.split_whitespace().collect();

    if parts.len() < 3 || parts[0] != "CONNECT" {
        anyhow::bail!("Expected CONNECT, got: {}", first_line);
    }

    // parts[1] is "example.com:443"
    let host = parts[1].split(':').next().unwrap_or("").to_string();
    Ok(host)
}

/// Reply 200 to establish the tunnel, then hand stream to TLS
async fn handle_connect(
    mut stream: TcpStream,
    ca:         Arc<CertificateAuthority>,
    cert_cache: Arc<CertCache>,
    client_cfg: Arc<rustls::ClientConfig>,
) -> Result<()> {
    // 1. Read CONNECT and extract hostname
    let hostname = read_connect_target(&mut stream).await?;
    tracing::info!("CONNECT to {hostname}");

    // 2. Send 200 Connection Established
    stream
        .write_all(b"HTTP/1.1 200 Connection Established\r\n\r\n")
        .await?;

    // 3. Generate (or fetch cached) leaf cert for this hostname
    let (cert_der, key_der) = cert_cache.get_or_generate(&hostname, &ca).await?;

    // 4. Build ServerConfig and accept TLS from client
    let server_cfg = Arc::new(build_server_config(cert_der, key_der)?);
    let client_tls = accept_tls(stream, server_cfg).await?;

    // 5. Connect upstream and TLS-handshake with real server
    let upstream_tcp = TcpStream::connect(format!("{}:443", hostname)).await?;
    let upstream_tls = connect_upstream_tls(upstream_tcp, &hostname, client_cfg).await?;

    // 6. Splice the two TLS streams — traffic flows bidirectionally
    proxy_streams(client_tls, upstream_tls).await?;

    Ok(())
}
```

## Layer 6 — Upstream TLS Re-encryption

After decrypting and optionally modifying traffic, you re-encrypt it toward the real server. You must **read and parse the response** before forwarding it to capture headers, status, and body.

### Proper Response Capture (Recommended)

```rust
use tokio::io::{AsyncReadExt, AsyncWriteExt};

async fn handle_mitm_request(
    mut client_tls: tokio_rustls::server::TlsStream<TcpStream>,
    mut upstream_tls: tokio_rustls::client::TlsStream<TcpStream>,
    request_data: Vec<u8>,
    app_handle: tauri::AppHandle,
) -> Result<()> {
    // 1. Forward request to upstream
    upstream_tls.write_all(&request_data).await?;

    // 2. Read complete response from upstream
    let mut response_buf = Vec::new();
    let mut body_offset = 0;
    let mut status_line = String::new();
    let mut response_headers = HashMap::new();

    loop {
        let mut read_buf = [0u8; 8192];
        match upstream_tls.read(&mut read_buf).await {
            Ok(0) => break,
            Ok(n) => {
                response_buf.extend_from_slice(&read_buf[..n]);

                // Parse headers on first pass
                if body_offset == 0 {
                    if let Some(header_end) = find_header_end(&response_buf) {
                        let header_str = String::from_utf8_lossy(&response_buf[..header_end]);
                        let lines: Vec<&str> = header_str.lines().collect();

                        if !lines.is_empty() {
                            status_line = lines[0].to_string();
                        }

                        for line in lines.iter().skip(1) {
                            let trimmed = line.trim();
                            if trimmed.is_empty() {
                                body_offset = header_end;
                                break;
                            }
                            if let Some((k, v)) = trimmed.split_once(':') {
                                response_headers.insert(k.trim().lowercase(), v.trim().to_string());
                            }
                        }

                        // Check if we have complete body
                        if let Some(cl) = response_headers.get("content-length")
                            .and_then(|v| v.parse::<usize>().ok())
                        {
                            if response_buf.len() >= header_end + cl {
                                response_buf.truncate(header_end + cl);
                                break;
                            }
                        } else if response_headers.get("transfer-encoding")
                            .map(|v| v.contains("chunked")).unwrap_or(false)
                        {
                            if response_buf.ends_with(b"0\r\n\r\n") {
                                break;
                            }
                        } else if response_buf.len() > header_end {
                            break;
                        }
                    }
                }

                // Safety limit
                if response_buf.len() > 50 * 1024 * 1024 {
                    break;
                }
            }
            Err(e) => break,
        }
    }

    // 3. Parse response status
    let (status, status_text, _) = parse_response_status(&response_buf);

    // 4. Extract body
    let body_offset = find_header_end(&response_buf).unwrap_or(0);
    let response_body = if body_offset < response_buf.len() {
        Some(String::from_utf8_lossy(&response_buf[body_offset..]).to_string())
    } else {
        None
    };

    // 5. Emit events for logging/UI
    let parsed_resp = ParsedResponse {
        status,
        status_text,
        headers: response_headers.clone().into_iter().map(|(k, v)| (k, v)).collect(),
        body: response_body.clone(),
        body_size: response_body.as_ref().map(|b| b.len()).unwrap_or(0),
        content_type: response_headers.get("content-type").cloned(),
    };
    emit_logger_response(&parsed_resp, &app_handle);

    // 6. Forward response to client
    client_tls.write_all(&response_buf).await?;
    client_tls.flush().await?;

    Ok(())
}
```

### Why NOT to Use Blind Splice

The naive approach of using `tokio::io::copy` for bidirectional splicing **cannot capture responses** because:

1. Bytes flow in both directions simultaneously
2. There's no point where you can pause and parse
3. Response data is immediately forwarded to client
4. Headers, status, and body are never extracted
5. The `api-call` event fires with `response_status: None`

### Response Parsing Helper

```rust
fn parse_response_status(buffer: &[u8]) -> (u16, String, usize) {
    let header_end = match find_header_end(buffer) {
        Some(pos) => pos,
        None => return (0, String::new(), 0),
    };

    let header_str = String::from_utf8_lossy(&buffer[..header_end]);
    let lines: Vec<&str> = header_str.lines().collect();

    if lines.is_empty() {
        return (0, String::new(), 0);
    }

    let parts: Vec<&str> = lines[0].split_whitespace().collect();
    let status = parts.get(1).and_then(|s| s.parse().ok()).unwrap_or(0);
    let status_text = parts.get(2).unwrap_or(&"").to_string();

    (status, status_text, header_end)
}
```

### Event Flow Summary

```
Client → Proxy (CONNECT)                    → 200 Connection Established
Client → Proxy (TLS handshake with fake cert)→ TLS OK
Proxy  → Upstream (TLS handshake with real) → TLS OK

Request Path:
  Client TLS → proxy_handler.request() → Parse headers/body
                                          → emit "logger-request"
                                          → emit "logger-curl"
                                          → emit "api-call" (request only)
                                          → Forward to upstream

Response Path:
  Upstream TLS → proxy_handler.response() → Read complete response
                                           → Parse status/headers/body
                                           → emit "logger-response"
                                           → emit "api-call" (with response)
                                           → Forward to client
```

## Full Proxy — Putting It All Together

```rust
use std::sync::Arc;
use anyhow::Result;
use tokio::net::{TcpListener, TcpStream};
use tokio::io::{AsyncReadExt, AsyncWriteExt};
use tokio_rustls::{TlsAcceptor, TlsConnector};
use rustls::{ServerConfig, ClientConfig, RootCertStore};
use rustls::pki_types::{CertificateDer, PrivateKeyDer, ServerName};
use rcgen::{Certificate, CertificateParams, DistinguishedName, IsCa, BasicConstraints, KeyPair};
use webpki_roots::TLS_SERVER_ROOTS;
use std::collections::HashMap;
use tokio::sync::Mutex;

// ─── Root CA ────────────────────────────────────────────────────────────────

struct Ca {
    cert: Certificate,
    key:  KeyPair,
}

impl Ca {
    fn new() -> Result<Self> {
        let mut params = CertificateParams::default();
        params.is_ca = IsCa::Ca(BasicConstraints::Unconstrained);
        let mut dn = DistinguishedName::new();
        dn.push(rcgen::DnType::CommonName, "My MITM CA");
        params.distinguished_name = dn;
        let key  = KeyPair::generate()?;
        let cert = params.self_signed(&key)?;
        Ok(Self { cert, key })
    }
}

// ─── Cert Cache ─────────────────────────────────────────────────────────────

struct CertCache(Mutex<HashMap<String, (Vec<u8>, Vec<u8>)>>);

impl CertCache {
    fn new() -> Arc<Self> {
        Arc::new(Self(Mutex::new(HashMap::new())))
    }

    async fn get(&self, host: &str, ca: &Ca) -> Result<(Vec<u8>, Vec<u8>)> {
        let mut m = self.0.lock().await;
        if let Some(v) = m.get(host) { return Ok(v.clone()); }

        let mut params = CertificateParams::new(vec![host.to_string()])?;
        let mut dn = DistinguishedName::new();
        dn.push(rcgen::DnType::CommonName, host);
        params.distinguished_name = dn;

        let lk   = KeyPair::generate()?;
        let lc   = params.signed_by(&lk, &ca.cert, &ca.key)?;
        let pair = (lc.der().to_vec(), lk.serialize_der());
        m.insert(host.to_string(), pair.clone());
        Ok(pair)
    }
}

// ─── TLS helpers ────────────────────────────────────────────────────────────

fn server_config(cert: Vec<u8>, key: Vec<u8>) -> Result<Arc<ServerConfig>> {
    let cfg = ServerConfig::builder()
        .with_no_client_auth()
        .with_single_cert(
            vec![CertificateDer::from(cert)],
            PrivateKeyDer::try_from(key).map_err(|e| anyhow::anyhow!("{e}"))?,
        )?;
    Ok(Arc::new(cfg))
}

fn client_config() -> Arc<ClientConfig> {
    let mut roots = RootCertStore::empty();
    roots.extend(TLS_SERVER_ROOTS.iter().cloned());
    Arc::new(ClientConfig::builder().with_root_certificates(roots).with_no_client_auth())
}

// ─── Response parsing helpers ───────────────────────────────────────────────

fn find_header_end(buffer: &[u8]) -> Option<usize> {
    for i in 0..buffer.len().saturating_sub(3) {
        if buffer[i] == b'\r' && buffer[i+1] == b'\n' &&
           buffer[i+2] == b'\r' && buffer[i+3] == b'\n' {
            return Some(i + 4);
        }
    }
    None
}

fn parse_response_status(buffer: &[u8]) -> (u16, String, usize) {
    let header_end = match find_header_end(buffer) {
        Some(pos) => pos,
        None => return (0, String::new(), 0),
    };

    let header_str = String::from_utf8_lossy(&buffer[..header_end]);
    let lines: Vec<&str> = header_str.lines().collect();

    if lines.is_empty() {
        return (0, String::new(), 0);
    }

    let parts: Vec<&str> = lines[0].split_whitespace().collect();
    let status = parts.get(1).and_then(|s| s.parse().ok()).unwrap_or(0);
    let status_text = parts.get(2).unwrap_or(&"").to_string();

    (status, status_text, header_end)
}

// ─── Connection handler ─────────────────────────────────────────────────────

async fn handle(
    mut stream: TcpStream,
    ca:         Arc<Ca>,
    cache:      Arc<CertCache>,
    cli_cfg:    Arc<ClientConfig>,
    app_handle: tauri::AppHandle,
) -> Result<()> {
    // Read CONNECT
    let mut buf = vec![0u8; 4096];
    let n = stream.read(&mut buf).await?;
    let raw = std::str::from_utf8(&buf[..n])?;
    let host = raw.lines().next()
        .and_then(|l| l.split_whitespace().nth(1))
        .and_then(|hp| hp.split(':').next())
        .unwrap_or("").to_string();

    // Respond 200
    stream.write_all(b"HTTP/1.1 200 Connection Established\r\n\r\n").await?;

    // Generate leaf cert
    let (cert, key) = cache.get(&host, &ca).await?;

    // Accept TLS from client
    let acceptor   = TlsAcceptor::from(server_config(cert, key)?);
    let mut client_tls = acceptor.accept(stream).await?;

    // Connect upstream
    let upstream_tcp = TcpStream::connect(format!("{host}:443")).await?;
    let connector    = TlsConnector::from(cli_cfg);
    let server_name  = ServerName::try_from(host.clone())?;
    let mut upstream_tls = connector.connect(server_name, upstream_tcp).await?;

    // Read request from client
    let mut request_buf = Vec::new();
    loop {
        let mut read_buf = [0u8; 8192];
        match client_tls.read(&mut read_buf).await {
            Ok(0) => break,
            Ok(n) => {
                request_buf.extend_from_slice(&read_buf[..n]);
                if let Some(end) = find_header_end(&request_buf) {
                    let cl = parse_content_length(&request_buf[..end]);
                    if let Some(len) = cl {
                        if request_buf.len() >= end + len {
                            request_buf.truncate(end + len);
                            break;
                        }
                    } else {
                        break;
                    }
                }
            }
            Err(_) => break,
        }
    }

    // Emit request event
    let parsed_req = parse_request_for_logger(&request_buf, &app_handle);
    emit_logger_request(&parsed_req, &app_handle);

    // Forward request to upstream
    upstream_tls.write_all(&request_buf).await?;

    // ── Read complete response from upstream (NOT splice) ──
    let mut response_buf = Vec::new();
    let mut body_offset = 0;
    let mut response_headers = HashMap::new();

    loop {
        let mut read_buf = [0u8; 8192];
        match upstream_tls.read(&mut read_buf).await {
            Ok(0) => break,
            Ok(n) => {
                response_buf.extend_from_slice(&read_buf[..n]);

                if body_offset == 0 {
                    if let Some(header_end) = find_header_end(&response_buf) {
                        let header_str = String::from_utf8_lossy(&response_buf[..header_end]);
                        for line in header_str.lines().skip(1) {
                            let trimmed = line.trim();
                            if trimmed.is_empty() {
                                body_offset = header_end;
                                break;
                            }
                            if let Some((k, v)) = trimmed.split_once(':') {
                                response_headers.insert(k.trim().lowercase(), v.trim().to_string());
                            }
                        }

                        if let Some(cl) = response_headers.get("content-length")
                            .and_then(|v| v.parse::<usize>().ok())
                        {
                            if response_buf.len() >= header_end + cl {
                                response_buf.truncate(header_end + cl);
                                break;
                            }
                        } else if response_headers.get("transfer-encoding")
                            .map(|v| v.contains("chunked")).unwrap_or(false)
                        {
                            if response_buf.ends_with(b"0\r\n\r\n") {
                                break;
                            }
                        } else if response_buf.len() > header_end {
                            break;
                        }
                    }
                }

                if response_buf.len() > 50 * 1024 * 1024 {
                    break;
                }
            }
            Err(_) => break,
        }
    }

    // Parse and emit response
    let (status, status_text, _) = parse_response_status(&response_buf);
    let body_offset = find_header_end(&response_buf).unwrap_or(0);
    let response_body = if body_offset < response_buf.len() {
        Some(String::from_utf8_lossy(&response_buf[body_offset..]).to_string())
    } else {
        None
    };

    let parsed_resp = ParsedResponse {
        status,
        status_text,
        headers: response_headers.clone().into_iter().map(|(k, v)| (k, v)).collect(),
        body: response_body.clone(),
        body_size: response_body.as_ref().map(|b| b.len()).unwrap_or(0),
        content_type: response_headers.get("content-type").cloned(),
    };
    emit_logger_response(&parsed_resp, &app_handle);

    // Forward response to client
    client_tls.write_all(&response_buf).await?;
    client_tls.flush().await?;

    // Emit api-call with full request+response
    let api_call = create_api_call(request_buf, response_buf, &response_headers);
    let _ = app_handle.emit("api-call", &api_call);

    Ok(())
}

fn parse_content_length(headers_data: &[u8]) -> Option<usize> {
    let headers_str = String::from_utf8_lossy(headers_data);
    for line in headers_str.lines() {
        let lower = line.to_lowercase();
        if lower.starts_with("content-length:") {
            return lower.trim_start_matches("content-length:").trim().parse().ok();
        }
    }
    None
}
```

### Key Differences from Splice-Based Proxy

| Approach | Request Capture | Response Capture | Modification |
|----------|----------------|------------------|--------------|
| **Splice** | ❌ None | ❌ None | ❌ Impossible |
| **This Implementation** | ✅ Full | ✅ Full | ✅ Possible |

The splice approach cannot be salvaged — it fundamentally cannot capture response data because bytes flow bidirectionally without pause points. Any proxy that needs to show traffic in a UI or modify requests/responses must read responses completely before forwarding.

## Implementation Summary

### The Splice Problem

The original MITM proxy approach used `tokio::io::copy` for bidirectional byte splicing:

```rust
// ❌ WRONG - Cannot capture responses
tokio::try_join!(
    tokio::io::copy(&mut client_read, &mut server_write),
    tokio::io::copy(&mut server_read, &mut client_write),
)?;
```

**Why this fails:**
- Bytes flow in both directions simultaneously
- No point to pause and parse response
- Response data is immediately forwarded to client
- `api-call` events emit with `response_status: None`
- Response headers, body, and status are never captured

### The Correct Approach

```rust
// ✅ CORRECT - Response capture before forwarding

// 1. Forward request to upstream
upstream_tls.write_all(&request_data).await?;

// 2. Read COMPLETE response from upstream
let mut response_buf = Vec::new();
loop {
    let n = upstream_tls.read(&mut read_buf).await?;
    if n == 0 { break; }
    response_buf.extend_from_slice(&read_buf[..n]);
    // Check if body is complete (Content-Length or chunked)
}

// 3. Parse response status, headers, body
let (status, status_text, body_offset) = parse_response_status(&response_buf);

// 4. Emit events
emit_logger_response(&parsed_resp, &app_handle);
emit_api_call(&api_call_with_response, &app_handle);

// 5. Forward response to client
client_tls.write_all(&response_buf).await?;
```

### Key Implementation Points

| Component | Purpose |
|-----------|---------|
| `find_header_end()` | Locate `\r\n\r\n` separator in raw buffer |
| `parse_content_length()` | Extract `Content-Length` from headers |
| `parse_response_status()` | Extract status code and text |
| Response loop | Read until body complete (CL or chunked or connection close) |
| Event emission | `logger-request` → forward → `logger-response` → `api-call` |

### Event Flow

```
1. Client CONNECT → proxy
2. Proxy 200 → Client
3. Client TLS handshake (fake cert) → Proxy
4. Proxy TLS handshake (real cert) → Server
5. Client HTTP request → Proxy → emit "logger-request"
6. Proxy HTTP request → Server
7. Server HTTP response → Proxy → emit "logger-response"
8. Proxy HTTP response → Client → emit "api-call" (full data)
```

## Security Notes & Legal Disclaimer

### How clients are deceived

The entire attack rests on one precondition: **the attacker's root CA must be trusted by the client**. Without it, the browser will show a certificate error and the connection will fail. Installing a CA requires:

- Physical access to the machine, OR
- Administrative privileges, OR
- Social engineering / MDM policy (enterprise environments)

### Legitimate use cases

- **Corporate TLS inspection** — enterprises install a CA via MDM to inspect employee traffic
- **Security testing tools** — mitmproxy, Charles Proxy, Burp Suite all work this way
- **Mobile app security research** — install CA on a test device to inspect API calls
- **Automated testing** — intercept HTTP in integration tests

### Detection indicators

Sophisticated clients can detect MITM by:

- **Certificate Transparency (CT) logs** — the fake cert won't appear in public CT logs
- **HPKP (HTTP Public Key Pinning)** — the app pins the real cert's public key
- **Certificate pinning** — mobile apps embed the expected cert fingerprint in code
- **TLS fingerprinting (JA3)** — the proxy's TLS ClientHello differs from the browser's
